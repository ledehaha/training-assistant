# 非学历培训全周期助手

## 一、项目背景与规划

### 1.1 项目背景

非学历培训作为高校继续教育的重要组成部分，涉及项目申报、方案设计、师资配置、场地安排、费用核算、满意度调查等多个环节。传统管理模式存在流程分散、数据孤岛、人工重复录入等问题，严重影响培训管理效率。

本项目旨在构建一套**全流程数字化、智能化**的培训管理平台，实现从需求分析到项目总结的完整闭环管理，帮助培训管理人员提升工作效率，降低管理成本。

### 1.2 建设目标

- **流程标准化**：统一培训项目全生命周期管理流程
- **数据集中化**：打破数据孤岛，实现数据共享与复用
- **操作智能化**：引入AI能力，辅助决策与数据处理
- **权限精细化**：支持多用户、多部门、多角色协同管理

### 1.3 技术架构

采用现代化的全栈技术方案：

| 技术层 | 技术选型 |
|--------|----------|
| 前端框架 | Next.js 16 + React 19 |
| UI组件 | shadcn/ui + Tailwind CSS 4 |
| 后端运行时 | Node.js 24 |
| 数据库 | sql.js（嵌入式SQLite） |
| AI服务 | 豆包/DeepSeek大模型 |
| 文件存储 | S3兼容对象存储 |
| 部署方式 | Docker容器化部署 |

## 二、核心功能模块

### 2.1 项目设计模块

支持培训项目的全流程设计：

- **需求分析**：记录培训目标、参训对象、预算范围
- **方案设计**：课程安排、师资配置、场地选择
- **费用预算**：自动计算讲师费、场地费、餐饮费等各项费用
- **智能推荐**：根据培训目标自动推荐匹配的课程、讲师、场地

### 2.2 项目申报模块

实现规范化的项目审批流程：

- **材料生成**：自动生成项目申报书、合同模板
- **审批流程**：学院提交 → 法务部+财务处同时审核 → 教务处终审
- **状态追踪**：实时查看审批进度与意见反馈

### 2.3 项目总结模块

提供完整的项目归档能力：

- **文件上传**：支持合同、成本测算表、申报书、学员名单、满意度调查等五类必要文件
- **AI数据检查**：智能解析上传文件，自动提取并校验关键数据
- **一键归档**：满足条件的项目可快速归档，生成完整档案

### 2.4 项目查询模块

提供多维度的项目检索：

- **状态筛选**：草稿、执行中、已完成、已归档
- **关键词搜索**：按项目名称、培训目标快速定位
- **统计分析**：培训人次、课时数、费用汇总等数据可视化

### 2.5 数据管理模块

维护培训业务核心数据：

- **讲师信息**：职称、专业领域、课时费标准、授课评价
- **场地信息**：位置、容量、设施、租金
- **课程信息**：统一的课程库，支持模板与项目课程
- **参访基地**：企业、政府部门等参观学习基地
- **规范性文件**：政策法规、费用标准等参考文档

### 2.6 用户管理模块

支持多用户协同管理：

- **部门管理**：区分管理部门（教务处、财务处等）与学院
- **角色权限**：系统管理员、部门负责人、学院员工等
- **审批配置**：不同角色对应不同的审批权限

## 三、智能模块亮点

### 3.1 AI智能推荐

基于培训需求智能推荐资源配置：

**推荐逻辑**：
- 分析培训目标、参训对象、预算范围
- 匹配课程库中的优质课程模板
- 推荐专业领域匹配的讲师
- 筛选满足人数要求的场地

**应用场景**：
> 管理员创建"中层管理干部能力提升"培训项目，系统自动推荐《领导力提升》《团队管理》等课程，匹配管理学领域的教授级讲师，并推荐容纳50人的培训教室。

### 3.2 AI智能导入

支持Excel/Word/PDF等多格式数据智能解析：

**核心能力**：
- 自动识别文件类型并解析结构
- 智能提取关键字段（姓名、职称、单位等）
- 自动校验数据格式与完整性
- 职称智能识别并匹配课时费标准
- 课时自动折算（分钟→课时）

**职称识别规则**：
| 职称级别 | 课时费标准 | 示例 |
|----------|-----------|------|
| 院士级 | 1500元/课时 | 院士、中科院院士 |
| 教授级 | 1000元/课时 | 教授、研究员、正高级工程师 |
| 其他 | 500元/课时 | 副教授、讲师、工程师等 |

### 3.3 AI归档检查

项目归档前智能检查与数据提取：

**检查流程**：
1. 解析合同文件 → 提取合同金额、签订日期
2. 解析成本测算表 → 校验费用明细
3. 解析项目申报书 → 核对项目基本信息
4. 解析学员名单 → 提取参训人数
5. 解析满意度调查 → 计算满意度评分

**智能提醒**：
- 自动发现数据不一致（如申报人数与实际参训人数不符）
- 提示缺失的必要文件
- 建议需要补充或修正的数据

### 3.4 AI总结报告

自动生成项目总结报告：

**报告内容**：
- 项目基本情况概述
- 培训目标完成情况分析
- 课程执行情况汇总
- 费用使用情况说明
- 满意度调查结果分析
- 改进建议与后续计划

## 四、后期规划

### 4.1 功能增强

- **移动端适配**：开发微信小程序，支持移动端审批与查询
- **消息通知**：集成企业微信/钉钉，实时推送审批通知
- **数据导出**：支持生成Word/PDF格式的正式文档
- **批量操作**：支持批量导入、批量审批、批量归档

### 4.2 智能升级

- **智能排课**：根据课程时长、讲师档期自动生成最优课表
- **风险预警**：自动识别预算超支、时间冲突等风险
- **趋势分析**：基于历史数据预测培训需求趋势
- **知识图谱**：构建培训领域知识图谱，提升推荐精准度

### 4.3 生态拓展

- **API开放**：提供标准化API接口，支持与第三方系统对接
- **数据互通**：与学校教务系统、财务系统数据打通
- **资源共建**：支持多校区、多学院资源共享
